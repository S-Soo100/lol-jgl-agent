# lol-jgl-agent 기획서

> 리그오브레전드 경기를 플레이한 후, 경기 데이터를 분석해
> **정글러에게 맞춤형 피드백과 조언**을 제공하는 개인 코칭 도구.

- 문서 버전: v0.1 (초안)
- 작성일: 2026-07-06
- 플랫폼: Windows 10+ (전용)
- 언어/스택: Python
- 조언 생성: 구독 Claude(Claude Code CLI) → 추후 API 교체 가능

---

## 1. 배경 & 문제 정의

- 대부분의 플레이어는 경기가 끝나면 **결과 화면(KDA, CS)만 보고 넘어간다.**
  왜 졌는지, 무엇을 고쳐야 하는지 스스로 분석하기 어렵다.
- 기존 전적 사이트(OP.GG 등)는 **통계는 주지만 "그래서 뭘 해야 하나"라는 조언은 주지 않는다.**
- 특히 **정글**은 동선·갱킹 타이밍·오브젝트 컨트롤 등 판단 요소가 많아
  초보~중급자가 혼자 개선하기 가장 어려운 포지션이다.

**→ 경기 데이터를 자동 분석해서, 정글러가 다음 판에 바로 적용할 수 있는
구체적 조언을 자연어로 주는 도구를 만든다.**

---

## 2. 목표

### 제품 목표
- 경기 종료 후 **1분 이내**에 "이번 판 정글 피드백 리포트"를 받는다.
- 리포트는 **숫자(지표) + 해석(강점/약점) + 실행 가능한 조언** 3층 구조.

### 개인/학습 목표 (이 프로젝트를 만드는 이유)
- Riot API·타임라인 데이터 분석 파이프라인 경험.
- LLM(Claude)을 분석 결과 위에 얹어 "조언 생성기"로 활용하는 패턴 학습.

### 비목표 (이번엔 안 함)
- 모든 포지션 지원 (정글 우선, 그 외는 후순위).
- 실시간 인게임 오버레이 (로드맵 4단계로 미룸).
- 계정/랭킹/소셜 기능, 수익화.
- `.rofl` 리플레이 파일 직접 파싱 (기술적으로 비현실적).

---

## 3. 타겟 사용자

- **1차: 나 자신 (정글러, 실력 향상 목적).** 도그푸딩으로 가치 검증.
- **2차: 정글을 배우는 아이언~플래티넘 구간 플레이어.**
- 페르소나: "매판 비슷하게 지는데 뭐가 문제인지 모르겠다. 코치는 부담스럽고,
  전적 사이트 숫자는 봐도 해석을 못 하겠다."

---

## 4. 핵심 사용자 시나리오

1. 게임을 한 판 끝낸다.
2. 도구를 실행하고 자기 Riot ID(`이름#태그`)를 입력한다. *(추후 자동 감지)*
3. 도구가 최근 경기의 결과 + 타임라인을 가져와 정글 지표를 계산한다.
4. 지표를 티어 기준과 비교해 강점/약점을 뽑고, Claude가 조언 문장을 생성한다.
5. 사용자는 **"이번 판 리포트"**(터미널/마크다운, 추후 이미지·웹)를 읽고
   다음 판에 고칠 점 1~3개를 얻는다.

## 5. 기능 명세

### 5.1 MVP (1단계) — 경기 후 리포트
| # | 기능 | 설명 | 우선순위 |
|---|------|------|:---:|
| F1 | 소환사 조회 | Riot ID → PUUID → 최근 매치 목록 | P0 |

| F4 | 벤치마크 비교 | 티어별 기준값과 대조해 등급 부여 | P1 |

## 6. 분석 지표 정의 (정글 특화)

> 상세 계산 로직은 `DESIGN.md` 및 `analysis/` 모듈. 여기서는 "무엇을 왜 보는가".

| 카테고리 | 지표 | 왜 중요한가 |
|---|---|---|
| **동선** | 첫 클리어 완료 시각, 풀/역클리어 여부, 첫 갱 타이밍 | 초반 템포·성장 효율의 핵심 |
| **성장** | CS@10/15, 분당 CS, 상대 정글러 대비 골드·경험치 격차 | 뒤처짐/앞섬을 객관화 |
| **갱킹** | 라인별 킬·어시 관여, 시간대별 분포, 관여 대비 성공률 | 정글의 팀 기여도 |
| **오브젝트** | 스폰 30초 전 위치, 참여율, 선점률 (드래곤/전령/바론) | 승패에 직결되는 자원 |
| **시야** | 와드 설치·제거, 제어와드, 오브젝트 주변 시야 장악 | 판단의 정보 기반 |
| **데스** | 위치 히트맵, 카정 데스, 오브젝트 연관 데스 | 반복되는 실수 패턴 발견 |
| **팀 비교** | 팀 내 딜량·골드·시야 점유율 | 캐리/폿수 여부 판단 |

각 지표는 **가능하면 티어 벤치마크와 비교**하여 좋음/보통/나쁨으로 표시하고,
LLM 조언의 근거로 넘긴다.

## 7. 시스템 개요 (요약)

```
Riot ID 입력
   → Riot API (Match + Timeline) 수집·캐시
   → 정글 지표 계산 + 벤치마크 비교
   → 강점/약점 요약 → Claude(구독) 조언 생성
   → 마크다운 리포트 출력
```

- 상세 아키텍처·폴더 구조·의존성은 **`DESIGN.md`** 참조.
- 조언 백엔드는 `SubscriptionAdvisor`(구독 CLI) ↔ `ApiAdvisor`(API 키) 교체 가능.

## 9. 리스크 & 대응

| 리스크 | 영향 | 대응 |
|---|---|---|
| Riot 개발용 키 24h 만료 / 낮은 rate limit | 개발 중 호출 실패 | 응답 로컬 캐시, 만료 시 재발급 안내, 지수 백오프 |
| 타임라인이 60초 단위라 세밀한 순간 포착 한계 | 분석 정밀도 | 이벤트(킬/오브젝트)는 초 단위 제공 → 위치는 보간 처리 |
| 정글 "잘함"의 기준이 챔프·메타마다 다름 | 조언 부정확 | 티어 벤치마크 + LLM 맥락 해석으로 보완, 절대 규칙 최소화 |
| 구독 Claude CLI 호출 방식 의존 | 백엔드 변경 리스크 | advisor 인터페이스 추상화로 API 백엔드 교체 가능 |
| 환경(파이썬 미설치, 홈디렉토리 git 얽힘) | 초기 세팅 지연 | M0에서 우선 해결 (아래 §10) |

## 10. 지금 당장 해결할 세팅 이슈 (M0)

1. **독립 git 저장소 분리** — 현재 `lol-jgl-agent`가 홈 디렉토리 git에 얽혀 있음.
   `lol-jgl-agent`에서 별도 `git init` 후 진행.
2. **Python 정식 설치** — 현재 PATH의 python은 MS Store 스텁, `py` 런처 경로 깨짐.
   python.org 또는 winget/uv로 3.11+ 설치.
3. **Claude CLI 경로 확보** — 조언 백엔드가 `claude -p`를 호출하려면 셸에서 실행 가능해야 함.
